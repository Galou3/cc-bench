# Examples / Exemples

## English

[`sample-report.md`](sample-report.md) and [`sample-report.csv`](sample-report.csv)
are a real report generated by the bundled mock agent:

```bash
ccbench run --suite suites/sample --conditions conditions --agent mock --reps 30 --out runs
ccbench report runs/latest --conditions conditions --out examples/sample-report.md
```

It shows the two outcomes that matter:
- `with-claude-md` → **+30.0%** (95% CI excludes 0, p = 0.0001): a *proven*
  improvement.
- `bloated-context` → −13.3% with raw p = 0.046 but a CI that touches 0:
  **not proven**. cc-bench refuses to overclaim a borderline effect.

Remember: the mock uses *injected* probabilities, so this proves the harness can
*detect* an effect — not anything about a real agent. Use `--agent claude` for real.

### FAQ
- **Is this a claim that a `CLAUDE.md` helps?** No. The mock demo proves the
  *method*. To make a real claim, run `--agent claude` on your own tasks.
- **Why "not proven" when p < 0.05?** Significance requires the bootstrap
  difference CI to exclude 0 *and* the corrected p to clear α. The bloated-context
  CI included 0, so the honest verdict is "not proven".
- **Does it cost anything?** The mock pipeline is free and offline. Only the real
  `claude`/`codex` adapters spend tokens.

## Français

[`sample-report.md`](sample-report.md) est un vrai rapport produit par l'agent
*mock* embarqué (sans clé API, sans coût) :

```bash
ccbench run --suite suites/sample --conditions conditions --agent mock --reps 30 --out runs
ccbench report runs/latest --conditions conditions --out examples/sample-report.md
```

Il montre les deux cas qui comptent :
- `with-claude-md` → **+30,0 %** (l'IC à 95 % exclut 0, p = 0,0001) : amélioration
  *prouvée*.
- `bloated-context` → −13,3 % avec p brut = 0,046 mais un IC qui touche 0 :
  **non prouvé**. `cc-bench` refuse de surinterpréter un effet limite.

À retenir : le mock utilise des probabilités *injectées* ; cela prouve que le
harness sait *détecter* un effet — pas quoi que ce soit sur un vrai agent. Pour du
réel : `--agent claude`.

### FAQ
- **Est-ce une preuve qu'un `CLAUDE.md` aide ?** Non. La démo mock prouve la
  *méthode*. Pour une vraie affirmation, lance `--agent claude` sur tes tâches.
- **Pourquoi « non prouvé » alors que p < 0,05 ?** La significativité exige que
  l'IC bootstrap de la différence exclue 0 *et* que la p corrigée passe α. Ici l'IC
  incluait 0, donc le verdict honnête est « non prouvé ».
- **Est-ce que ça coûte quelque chose ?** Le pipeline mock est gratuit et hors
  ligne. Seuls les adaptateurs réels `claude`/`codex` consomment des tokens.
